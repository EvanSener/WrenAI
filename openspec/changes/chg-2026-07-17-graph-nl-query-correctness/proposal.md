# 为什么

真实广告问题“最近 15 天，按租户、站点、广告活动和广告组统计推广品曝光、点击和
花费”暴露了四个相互独立的 Graph 自然语言问题：

- “最近 N 天”没有转换为增量事实表的明确分区范围；
- 整句出现的业务对象被错误加入显式 `按...统计` 的分组维度；
- 通用 Cube 描述中的“曝光/点击/花费”会压过明确的“推广品”业务主题，选错事实；
- 事实表已有的主数据关系键仍被强制回查最新快照，产生冗余 Join 和空键。

# 变更内容

- `graph query --question ... --execute` 对 MaxCompute 的“最近/过去 N 天”内部执行
  一次 `max_pt` 探测，再把精确闭区间交给原 Graph Planner。
- 显式 `按/by` 短语只控制分组维度；短语外业务对象继续参与事实源召回。
- 事实源语义证据优先于 Cube 通用指标文案，Cube priority 仍只用于后置平局裁决。
- 当事实本地维度字段正是到 `master_model` 的安全关系键时直接投影本地键；描述性
  主数据属性仍使用权威模型。

# 不做什么

- 不让编译模式访问数据库；相对最新分区窗口只在 `--execute` 中解析。
- 不放宽 M:1/1:1、Fanout、M:N、Grain、Additivity 或只读安全规则。
- 不把事实本地描述字段当作主数据；本地优化只适用于经过关系条件验证的身份键。
